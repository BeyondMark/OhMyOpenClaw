# 2026-03-09 OpenClaw 升级后 gateway 无限重启修复笔记

## 背景
- 升级到 `OpenClaw 2026.3.8` 后，通过 `curl -fsSL https://openclaw.ai/install.sh | bash` 安装。
- 执行 `openclaw gateway status` 时表面上能看到 LaunchAgent、端口和探活信息，但后台服务实际出现过一段持续重启。

## 现象
- `launchd` 反复拉起 `ai.openclaw.gateway`。
- `~/.openclaw/logs/gateway.err.log` 持续出现同一条报错：

```text
Gateway start blocked: set gateway.mode=local (current: unset) or pass --allow-unconfigured.
```

- `/tmp/openclaw/openclaw-2026-03-09.log` 中还能看到伴随症状：

```text
gateway connect failed: Error: gateway closed (1008):
```

## 根因
- 新版本 gateway 启动时要求配置文件中显式存在 `gateway.mode=local`，否则服务会被阻止启动。
- LaunchAgent 本身带有 `keepalive`，所以一旦 gateway 因配置不满足条件而退出，就会表现为“无限重启”。
- 官方 CLI 文档也明确说明：如果未配置 `gateway.mode=local`，只有显式传 `--allow-unconfigured` 才允许启动；排障文档则建议重新设置本地模式。

## 本机排查过程
1. 查看服务状态：

```bash
openclaw gateway status
openclaw status
launchctl print gui/$(id -u)/ai.openclaw.gateway
```

2. 查看网关错误日志，定位真正阻塞原因：

```bash
sed -n '1,120p' ~/.openclaw/logs/gateway.err.log
```

3. 核对配置文件：

```bash
sed -n '1,220p' ~/.openclaw/openclaw.json
```

4. 确认缺失项就是 `gateway.mode`，而不是端口、token 或 LaunchAgent 安装问题。

## 修复动作
按官方支持的配置命令直接修：

```bash
openclaw config set gateway.mode local
openclaw config validate
openclaw gateway restart
```

如果希望走向导方式，也可以用官方排障文档提到的方式重新写入本地模式：

```bash
openclaw setup --mode local
```

## 修复后验证
- `~/.openclaw/openclaw.json` 已包含：

```json
"gateway": {
  "mode": "local",
  "auth": {
    "mode": "token"
  }
}
```

- `openclaw status` 显示：
  - `Gateway: local`
  - `Gateway service: LaunchAgent installed · loaded · running`
- `~/.openclaw/logs/gateway.log` 显示 gateway 成功监听 `ws://127.0.0.1:18789`
- 重启后再次检查 `launchctl print gui/$(id -u)/ai.openclaw.gateway`，未再出现持续递增的异常拉起现象。

## 结论
- 这次故障不是安装脚本本身损坏，而是升级后的新要求和旧配置状态不兼容。
- 真正要修的是配置文件中的 `gateway.mode`，不是单纯反复重装或只看 `gateway status` 输出。
- 以后升级 OpenClaw 后如果出现 gateway 异常重启，优先先看 `~/.openclaw/logs/gateway.err.log`，再确认 `gateway.mode` 是否已显式写成 `local`。
